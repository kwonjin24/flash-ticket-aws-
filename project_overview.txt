Flash Tickets 프로젝트 소개
---------------------------
Flash Tickets는 급격한 트래픽이 몰리는 대규모 이벤트 티켓팅 상황에서 오버셀 없이 주문을 처리하고, 실시간 대기열 상태를 사용자에게 투명하게 제공하기 위해 설계된 서비스입니다. 하나의 모노레포 안에서 NestJS 기반 API, Vite React 프런트엔드, 모의 결제 워커를 함께 관리하며, 메시지 큐와 Redis를 조합한 대기열 승격 로직을 통해 사용자의 대기 상태와 입장 토큰 발급을 안정적으로 제어합니다. 운영자는 관측 및 로그 파이프라인을 통해 주문 흐름과 큐 상태를 상시 모니터링하고, 장애 상황에서도 빠르게 대응할 수 있습니다.

프로젝트 주요 구성 기술
------------------------
- 애플리케이션 레이어: NestJS 11 (TypeScript), BullMQ 워커, Socket.IO WebSocket 게이트웨이, class-validator/class-transformer 기반 DTO 검증.
- 프런트엔드: Vite + React 18 + TypeScript, Zustand 상태 관리, Socket.IO 클라이언트를 활용한 실시간 대기열 UI.
- 메시징/큐: Redis 기반 BullMQ 반복 작업으로 대기열 승격 스케줄링, RabbitMQ를 이용한 주문-결제 비동기 처리 및 모의 결제 응답 스트림.
- 데이터베이스: PostgreSQL (Amazon RDS) + TypeORM으로 주문/이벤트/사용자 스키마 및 트랜잭션 관리, Redis (Amazon ElastiCache)로 대기열 상태 캐싱 및 토큰 TTL 관리.
- 인프라 및 운영: Amazon EKS 기반 컨테이너 오케스트레이션, Amazon ECR 이미지 레지스트리, NGINX Ingress Controller와 AWS NLB/Route53/ACM으로 TLS 종료 및 트래픽 라우팅, Prometheus + Grafana + Loki(+S3)로 관측과 로그 수집, Cilium & Hubble로 eBPF 네트워크 보안과 가시성 확보.
- 배포/자동화: Docker 기반 멀티 스테이지 빌드, pnpm Workspace, GitHub Actions(또는 유사 CI) 파이프라인, Helm/Kustomize를 이용한 매니페스트 관리 옵션.
- 개발 도구: Node.js 20.19.x, pnpm 10.x, ESLint/Prettier, Jest 기반 유닛 테스트, ts-node-dev/tsc watch 개발 경험 최적화.
