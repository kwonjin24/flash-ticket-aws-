대기열 API와 워커를 구현하라.

파일:
- api/src/queue/queue.module.ts
- api/src/queue/queue.controller.ts
- api/src/queue/queue.service.ts
- api/src/queue/queue.processor.ts (BullMQ)
- api/src/queue/redis.provider.ts (ioredis 연결)
- api/src/queue/types.ts

요구사항:
- 엔드포인트:
  POST /queue/enqueue {eventId} (JWT 필요) → {ticketId}
  GET  /queue/status?ticketId=... → {state: QUEUED|READY|EXPIRED, position?, gateToken?}
  POST /queue/enter {ticketId, gateToken} (JWT) → 200 (1회성 사용 처리)
- Redis 구조:
  ZSET key: event:{eventId}:queue (score=enqueue_ts)
  HASH key: ticket:{ticketId} => {userId,eventId,state,gateToken?,expiresAt?}
- READY 발급: BullMQ 워커에서 주기적으로 capacity만큼 READY로 전환, gateToken(난수) 발급, TTL(예: 2m).
- enter 시 gateToken 검증/소모(상태 USED) 처리.
- 테스트 편의를 위한 capacity/env 변수 반영.
- 전체 코드 출력.
