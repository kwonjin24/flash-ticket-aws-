주문 생성(HOLD) API를 구현하라.

파일:
- api/src/orders/orders.module.ts
- api/src/orders/orders.controller.ts
- api/src/orders/orders.service.ts
- api/src/orders/dto/*.ts (CreateOrderDto)

요구사항:
- POST /orders 헤더: Authorization(Bearer), Idempotency-Key, X-Gate-Token
  바디: {eventId, qty}
  응답: {orderId, status:'HOLD', amount}
- 검증:
  - 게이트 토큰 유효성/1회성 확인(Queue 서비스에 의존).
  - 사용자당 maxPerUser 초과 금지.
  - idemKey 중복 시 이전 결과 재연.
- 트랜잭션:
  - HOLD 레코드 생성 (아직 재고 차감 X).
  - 딜레이 작업(예: 5분) enqueue하여 미결제 만료 처리 예정.
- 전체 코드 출력.
